\subsection subsec_buttons Basic Buttons

<p>
<ul>
<li> OK (Enter) - Accept changes and hide window.
<li> Cancel (Esc) - Drop changes and hide window.
<li> Apply - Accept changes and keep window opened.
<li> Load (CTRL + L) - Load settings from a file (*.ini).
<li> Save (CTRL + S) - Save settings in the current file (*.ini). When a file has not been loaded yet, the "Save As ..." file dialog is opened.
<li> Save As... (CTRL + D) - Save settings into a file (*.ini).
<li> Edit (CTRL + E) - Edit the peptide sequence using the [Draw Peptide Tool](@ref drawpeptide). The sequence is imported into the draw peptide tool and visualized if its syntax is correct

</ul>

\section settingssearch Search

\subsection settings_mode Mode

<p>
<ul>
<li><b>'De Novo Search Engine - MS/MS'</b> - perform the <i>de novo</i> sequencing.
<li><b>'Compare Peaklist(s) with Spectrum of Searched Sequence - MS/MS'</b> - a theoretical spectrum is generated from the input "Searched Sequence" and compared with the peaklist(s).
<li><b>'Compare Peaklist with Database - MS/MS'</b> - a peaklist is compared with theoretical spectra generated from a database of sequences.
<li><b>'Compare Peaklist(s) with Database - MS, LC-MS, MSI'</b> - dereplication; the peaklist(s) are compared with a theoretical peaklist generated from a database of compounds.
<li><b>'Compound Search - MS, LC-MS, MSI'</b> - the peaklist(s) are compared with a theoretical peaklist generated from a list of input chemical elements (a database-free search method).
</ul>
</p>

\subsection settings_charge Charge

<p>Enter the charge of precursor ion. Negative values are allowed. The charge of precursor ion in the input peaklist file is ignored.
If the mode "Compare Peaklist(s) with Database - MS or MSI data" is used, the value determines the maximum charge of generated theoretical peaks.
For example, 3 means that theoretical peaks of compouds are generated with charges 1+, 2+, and 3+. The value -1 means that theoretical
peaks of compounds are generated with charge 1-.</p>

\subsection settings_precursormz Precursor m/z Error Tolerance

<p>Enter the precursor m/z error tolerance in ppm.</p>

\subsection settings_fragmentmz m/z Error Tolerance

<p>Enter the m/z error tolerance in MS mode or the fragment m/z error tolerance in MS/MS mode [ppm].</p>

\subsection settings_relint Minimum Threshold of Relative Intensity

<p>Enter the minimum threshold of relative intensity in %. Peaks with relative intensities below the threshold are removed from the peaklist.</p>

\subsection settings_absint Minimum Threshold of Absolute Intensity

<p>Enter the minimum threshold of absolute intensity. Peaks with absolute intensities below the threshold are removed from the peaklist.</p>

\subsection settings_mz m/z Ratio

<p>Enter the minimum m/z ratio and maximum m/z ratio. Experimental peaks with m/z ratios below/above the thresholds are removed from the peaklist(s). For maximum m/z, 0 = disabled.</p>

\section settingsbrickdatabase Database of Building Blocks

\subsection settings_blocksdb Building Blocks Database File (CTRL + B)

<p>A text file containing a database of building blocks. See [Format of Building Blocks Databases](@ref blockseditordbformat).</p>

\subsection settings_maxblocks Maximum Number of Combined Blocks (start / middle / end)

<p>Maximum number of combined building blocks to skip a gap in a de novo graph. A small value speeds up the search and vice versa.
Depending on the position of a gap between peaks in a spectrum, the three different values are set up:</p>
<ul>
<li><b>Start:</b> Maximum number of combined building blocks to skip a gap leading <b>from a start point</b> in a de novo graph.
<li><b>Middle:</b> Maximum number of combined building blocks to skip a gap in the middle of the graph. <b>Except gaps leading from a start point or to an end point.</b>
<li><b>End:</b> Maximum number of combined building blocks to skip a gap leading <b>to an end point</b> in the de novo graph.
</ul>

<p>In the following scheme, beauverolide I is detected. Since the peak between m/z 1 and 304 is missing, the minimum values start/middle/end are 2/1/1.
Otherwise, beauverolide I is not detected when "Incomplete Paths in De Novo Graph" is set up to "keep" or "remove".</p>

<p>
\image html denovograph-v4.png "De novo graph created from mass spectrum of beauverolide I."
\image latex denovograph-v4.eps "De novo graph created from mass spectrum of beauverolide I." width=14cm
</p>

<p><b>Tip:</b> When an unknown molecule is analyzed, check peptide sequence tags first. Use the following settings:</p>
<ul>
<li><b>Set up start/middle/end to 1/1/1.</b>
<li><b>Set up the "Peptide Type" to "linear" or "cyclic".</b>
<li><b>Set up the "Incomplete Paths in De Novo Graph" to "connect".</b>
</ul>

<p>When a branched peptide is analyzed, the minimum values are 1/2/1 (the branch is in the middle), 1/1/3 (the branch is at the end) or 3/1/1 (the branch is at the beginning). If smaller values are used, the branch cannot be detected. See also [Branched Sequence Detection](@ref brancheddetection).</p>

<p>When a branch-cyclic peptide is analyzed, the minimum values are 1/2/1, 1/1/2 or 2/1/1. See also [Branch-cyclic Sequence Detection](@ref branchcyclicdetection).</p>

\subsection settings_blindpaths Incomplete Paths in De Novo Graph

<p>The operation to be performed with edges forming incomplete paths (i.e., the paths which do not lead from the start node or do not lead to the end node). 
The following options can be selected:
<ul>
<li> keep (edges are kept - you can see a complete de novo graph in 'View -> Graph')
<li> remove (edges are removed - speeds up the search)
<li> connect (edges are connected - allow detection of sequence tags)
</ul>
</p>

\subsection settings_seqtag Peptide Sequence Tag

<p>Each peptide sequence candidate generated from a de novo graph must fulfil the peptide sequence tag. Otherwise, its theoretical spectrum is not generated and 
the peptide sequence candidate is excluded from the search. A name of a building block must be enclosed in '[' and ']'. Enclosed building blocks must be separated by '-'. 
A branch of a branched or a branch-cyclic peptide must be enclosed in "\(" and "\)". Additional backslashes must be used because of a special meaning of '(' and ')' in regular expressions. 
The tag can be a regular expression in <a href="http://www.cplusplus.com/reference/regex/ECMAScript/">ECMAScript syntax</a> except symbols '[' and ']' which 
have a special meaning and enclose a building block. Examples of peptide sequence tags:
<ul>
<li> <code>[Val]-[Lac]-[Val]-[Hiv]</code> (a peptide sequence candidate must contain the following subsequence: valine, lactic acid, valine, 2-hydroxyisovaleric acid)
<li> <code>([Val]-[Lac]-[Val]-[Hiv]-*){3}</code> (the tag must be repeated exactly 3x in a peptide sequence candidate, i.e., the tag corresponds to valinomycin; the symbol '*' means that the previous symbol '-' may or may not be present)
<li> <code>[Pro]-[Ile]-[Ile]\\([Orn]-[N-Ac-Ile]\\)[Phe]</code> (a tag corresponding to the branched peptide linearized pseudacyclin A)
</ul>
</p>

<p>
Note that a peptide sequence candidate is kept if the tag is detected in any linearized sequence of a cyclic, branched or branch-cyclic peptide sequence. See also [Format of Sequence Databases](@ref sequenceeditordbformat).
</p>

\subsection settings_maximumcombinedlosses Maximum Number of Combined Losses / Elements

<p>Maximum number of combined neutral losses or chemical elements.</p>

\subsection settings_reportunmatched Report Unmatched Theoretical Peaks

<p>If checked, all unmatched theoretical peaks are reported. If unchecked, unmatched theoretical peaks are reported only if a corresponding isotope pattern has been matched. 
This feature may spend a lot of main memory, keep it disabled if possible.</p>

\subsection settings_isotopepatterns Generate Full Isotope Patterns

<p>
The full isotope patterns of compounds are generated in theoretical spectra. The [FWHM](@ref settings_fwhm) value is used for this purpose. If checked, the deisotoping is disabled automatically.
</p>

\subsection settings_minimumpatternsize Minimum Pattern Size

<p>
The minimum number of matched peaks in an isotope pattern to be reported. The option "Generate Full Isotope Patterns" must be enabled.
</p>

\subsection settings_minimumfeaturesize Minimum Feature Size

<p>
The minimum number of consecutive scans in which a compound must be identified to be reported (LC-MS data only).
</p>

\subsection settings_minimumiontypes Minimum Ion Types

<p>
The minimum number of ion types which must be matched to report a given compound. Use e.g. Ion Types: [M+H]+, [M+Na]+; Charge: 2; Neutral Losses: H2O; Maximum Number of Combined Losses: 1.
If Minimum Ion Types: 2, then any pair of ions from the set of ions [M+H]+, [M+Na]+, [M+2H]2+, [M+Na+H]2+, [M+H-H2O]+, [M+Na-H2O]+, [M+2H-H2O]2+, [M+Na+H-H2O]2+ must be matched to report a given compound.  
</p>

\subsection settings_basicformulacheck Basic Formula Check

<p>
Apply Senior's filtering rules. See <a href="https://doi.org/10.1186/1471-2105-8-105">https://doi.org/10.1186/1471-2105-8-105</a> for more details.
</p>

\subsection settings_advancedformulacheck Advanced Formula Check

<p>
Advanced filtering rules are applied if molecular formulas of compounds are generated. See the tables 1, 2, and 3 in <a href="https://doi.org/10.1186/1471-2105-8-105">https://doi.org/10.1186/1471-2105-8-105</a> for more details.
</p>

\subsection settings_isotopicratio Isotopic Ratio Threshold

<p>
Check the ratios of intensities of matched peaks (0 = disabled).
</p>

\subsection settings_searchedseq Sequence

<p>A peptide sequence which you are searching for or a peptide sequence tag. A peptide sequence must be entered if "Mode" is set up to "Compare Peaklist(s) with Spectrum of Searched Sequence". 
Otherwise, the option is similar to "Peptide Sequence Tag" with a difference that a peptide sequence candidate is not removed from the search but it is just highlighted in the
output list of peptide sequence candidates. If "Compare Peaklist(s) with Spectrum of Searched Sequence" mode is used, the only possible format of the expression is a peptide sequence 
(e.g., <p><code>[Val]-[Lac]-[Val]-[Hiv]-[Val]-[Lac]-[Val]-[Hiv]-[Val]-[Lac]-[Val]-[Hiv]</code></p> or <p><code>[Pro]-[Ile]-[Ile]\\([Orn]-[N-Ac-Ile]\\)[Phe]</code></p>).
Regular expressions like <p><code>([Val]-[Lac]-[Val]-[Hiv]-*){3}</code></p> are not supported in this mode. See also [Format of Sequence Databases](@ref sequenceeditordbformat).
</p>

\subsection settings_ntermmodif N-terminal Modification

\subsection settings_ctermmodif C-terminal Modification

<p>A name of a C-terminal modification which belongs to the searched peptide. The name must be defined in [N-/C-terminal Modifications File](@ref settings_modif).</p>

\subsection settings_branchmodif Branch Modification

<p>A name of an N-terminal or C-terminal modification which belongs to a branch of a searched peptide (branched and branch-cyclic peptides only). The name must be defined in [N-/C-terminal Modifications File](@ref settings_modif).</p>
