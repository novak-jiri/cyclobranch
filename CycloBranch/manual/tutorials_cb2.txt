\section tutorial1_cb2 Tutorial 1: Dereplication in MS1 data

<p>The dereplication is the process of annotation of already known compounds (i.e. the database search). To perform the dereplication, click on [Search->Settings](@ref settings) in the main window of the application and configure the tool. 

<p>
\image html tutorial-2-1-mainwindow.png "Tutorial 1: Main window of the engine."
\image latex tutorial-2-1-mainwindow.png "Tutorial 1: Main window of the engine." width=16cm
</p>

<p>The most important parameters are:</p>

<p><b>Mode</b> - select [Compare Peaklist(s) with Database - MS, LC-MS, MSI](@ref settings_mode).</p>

<p><b>File</b> - select an input file with mass spectra. The list of supported file formats is available [here](@ref settings_peaklistfile) (you may need a third-party software for some file formats). 
Enable the checkbox, if you would like to extract profile mass spectra from the file for later visualization (note: the input file must include profile spectra; the extraction may be time-consuming; 
and the visualization must be enabled in the spectrum detail window).</p>

<p><b>Charge</b> - the maximum charge of generated theoretical peaks. If the value is equal to 2, then the theoretical peaks of singly and doubly charged ions are generated. Use -1 for negative mode.</p>

<p><b>FWHM</b> - see the profile peaks in your spectra and estimate the full width at half maximum (i.e. the width of a representative peak at half of his height; the value corresponds to m/z ratio of a peak divided by its resolution). 
The theoretical peaks are split or merged using FWHM value prior they are matched with experimental peaks.</p>

<p><b>Sequence/Compound Database File</b> - select a text file serving as a database. The proper text file can be prepared using  [Sequence/Compound Database Editor](@ref sequenceeditorinterface) or created using 
a <a href="https://ms.biomed.cas.cz/cyclobranch/release/custom_databases_excel_sheets.zip" onclick="ga('send', 'event', 'Sequences', this.href);">template in Excel</a>. Several [in-house prepared databases](@ref availablesequencedatabases)
are also available. In this mode, it is sufficient if you define only a name and a molecular formula (neutral) for every compound. The type of compound can be defined as 'other' 
and the monoisotopic mass is calculated from the molecular formula automatically. The text file has items separated by tabulator on each row and can also be easily prepared using a script (please, remember that you have 
to keep the correct number of columns/tabulators to make the file readable by the search engine).</p>

<p><b>Ion Types</b> - select the types of ions to be searched. Remember that for negative ions (e.g. [M-H]-), the value of charge must be negative (i.e. -1).</p>

<p><b>Generate Full Isotope Patterns</b> - enable the checkbox to generate the full isotopic patterns of compounds (the parameter FWHM must be estimated correctly). Optionally, you can disable this feature to generate only
the theoretical peaks of monoisotopic ions. It may be advantageous for a fast check of a dataset.</p>

<p><b>Minimum Number of Isotopic Peaks</b> - define the minimum number of peaks which must be matched in an isotopic pattern to report a given compound. The feature <b>Generate Full Isotope Patterns</b> must be enabled.</p>

<p>
\image html tutorial-2-1-settings.png "Tutorial 1: Configuration for dereplication in MS1 data."
\image latex tutorial-2-1-settings.png "Tutorial 1: Configuration for dereplication in MS1 data." width=16cm
</p>

<p>We recommend to save the configuration into an '.ini' file.</p>

<p>Click <b>OK</b>. In the main window select <b>Search->Run</b> to process the dataset.</p>

<p>
\image html tutorial-2-1-result.png "Tutorial 1: Search process completed."
\image latex tutorial-2-1-result.png "Tutorial 1: Search process completed." width=16cm
</p>

<p>Double-click on the row to display the details. Click on the highlighted icon to display the profile spectrum (if the option was enabled in the settings dialog).</p>

<p>
\image html tutorial-2-1-detail.png "Tutorial 1: Detail of a mass spectrum. The red peaks were matched."
\image latex tutorial-2-1-detail.png "Tutorial 1: Detail of a mass spectrum. The red peaks were matched." width=16cm
</p>

\section tutorial2_cb2 Tutorial 2: Dereplication in LC-MS data

<p>Follow the instructions in [Tutorial 1](@ref tutorial1_cb2) and use the following extra parameter:</p>

<p><b>Minimum Number of Spectra</b> - the minimum number of consecutive scans in which a compound must be matched to be reported.</p>

<p>
\image html tutorial-2-2-settings.png "Tutorial 2: Configuration for dereplication in LC-MS data."
\image latex tutorial-2-2-settings.png "Tutorial 2: Configuration for dereplication in LC-MS data." width=16cm
</p>

<p>All MS1 scans included in the input file are listed in the output report. Double-click on any row to display the spectrum from the corresponding scan.</p>

<p>
\image html tutorial-2-2-result.png "Tutorial 2: Search process completed."
\image latex tutorial-2-2-result.png "Tutorial 2: Search process completed." width=16cm
</p>

<p>In the main window, click on [Tools->Chromatogram](@ref chromatogram) to see the chromatographic peaks of detected compounds and their retention times. Double-click on any chromatographic peak to open the corresponding mass spectrum.</p>

<p>
\image html tutorial-2-2-chromatogram.png "Tutorial 2: Chromatogram."
\image latex tutorial-2-2-chromatogram.png "Tutorial 2: Chromatogram." width=16cm
</p>

<p>In the main window, click on [Tools->Summary Table of Matched Peaks](@ref summarypeakstable) to see the list of all matched peaks in all the scans. Double-click on any row to open the corresponding mass spectrum. 
The table can be filtered using any column e.g. <b>Name</b> or <b>Theoretical m/z</b>. Start typing into the text field and the names of identified compounds are automatically suggested.
Use the arrows to browse the identified compounds individually.</p>

<p>The table is interactively-connected with the chromatogram. What you see in the table is automatically sent to [Chromatogram](@ref chromatogram) window. So, you can easily browse extracted ion chromatograms of all identified compounds.</p>

<p>
\image html tutorial-2-2-summarytable.png "Tutorial 2: Summary Table of Matched Peaks."
\image latex tutorial-2-2-summarytable.png "Tutorial 2: Summary Table of Matched Peaks." width=16cm
</p>

\section tutorial3_cb2 Tutorial 3: Identification of unknown compounds in LC-MS data

<p>See the [Tutorial 1](@ref tutorial1_cb2) and [Tutorial 2](@ref tutorial2_cb2). In this example, any database of compounds is not used but the molecular formulas of compounds are generated from an input list
of chemical elements e.g. H, C, O, N, S, and P. Because the search space grows exponentially with increasing m/z value and the number of input elements, carefully configure the following parameters 
to limit the number of false-positive formulas reported, to speed-up the search process and to save the computer's main memory:</p>

<p><b>Mode</b> - select [Compound Search - MS, LC-MS, MSI](@ref settings_mode).</p>

<p><b>m/z Error Tolerance</b> - define m/z error tolerance e.g. 2 ppm. You can use bigger value e.g. 5 or 10 ppm but in this case define also the parameter <b>Isotope m/z Tolerance</b>.</p>

<p><b>Minimum Threshold of Relative Intensity [%]</b> - use e.g. 3 or 5% for an initial check of a dataset. Check the results and optionally reduce to 1% or lower value. Use 0 to disable this feature. The value is used together with <b>Minimum Threshold of Absolute Intensity</b>.</p>

<p><b>Minimum Threshold of Absolute Intensity</b> - use bigger value (e.g. 100000) for an initial check of a dataset and then optionally reduce to lower value. Use 0 to disable this feature. The value is used together with <b>Minimum Threshold of Relative Intensity</b>.</p>

<p><b>m/z Ratio</b> - define the range of m/z values in which the compounds should be detected. If a narrow interval is used (e.g. 100 or 200 Da), the search process is faster. If a wide interval (e.g. 1000 Da) is used, the searching is slower.</p>

<p><b>FWHM</b> - see the profile peaks in your spectra and estimate the full width at half maximum (i.e. the width of a representative peak at half of his height; the value corresponds to m/z ratio of a peak divided by its resolution). 
The theoretical peaks are split or merged using FWHM value prior they are matched with experimental peaks.</p>

<p><b>Ion Types</b> - select the types of ions to be searched. Remember that for negative ions (e.g. [M-H]-), the value of charge must be negative (i.e. -1). For an initial check of a dataset, use only one or a very small number of ions.</p>

<p><b>Neutral Losses / Chemical Elements</b> - Define an input list of chemical elements. Even if any custom element can be added, use the smallest number of elements as possible. Use the colon to limit the maximum number of occurrences 
of elements like sulfur or phosphorus (e.g. S:2 or P:1; double-click on an element to edit this value). If the ions like [M+Na]+, [M+K]+ and [M+Fe-2H]+ are searched, do not add the elements Na, K, and Fe, respectively. 
They are used automatically.</p>

<p><b>Maximum Number of Combined Losses / Elements</b> - define the maximum number of elements in a molecule (e.g. 150 or 200). Example, the ion [C39H60N6O15+H]+ has 39+60+6+15+1=121 elements.</p>

<p><b>Generate Full Isotope Patterns</b> - enable the checkbox to generate the full isotopic patterns of compounds (the parameter FWHM must be estimated correctly).</p>

<p><b>Minimum Number of Isotopic Peaks</b> - define the minimum number of peaks which must be matched in an isotopic pattern to report a given compound. The feature <b>Generate Full Isotope Patterns</b> must be enabled.
Use e.g. 2 if a big value of <b>Minimum Threshold of Relative Intensity [%]</b> is used.</p>

<p><b>Minimum Number of Spectra</b> - the minimum number of consecutive scans in which a compound must be matched to be reported.</p>

<p><b>Basic Formula Check</b> - apply Senior's filtering rules to remove unlikely molecular formulas. See <a href="https://doi.org/10.1186/1471-2105-8-105">https://doi.org/10.1186/1471-2105-8-105</a> for more details.</p>

<p><b>Advanced Formula Check</b> - advanced filtering rules are applied if molecular formulas of compounds are generated. See the tables 1, 2, and 3 in <a href="https://doi.org/10.1186/1471-2105-8-105">https://doi.org/10.1186/1471-2105-8-105</a> for more details.</p>

<p><b>N/O Ratio Check</b> - check if the number of nitrogen atoms is less or equal to the number of oxygen atoms.</p>

<p><b>Isotope m/z Tolerance</b> - maximum m/z difference between an isotopic peak and the monoisotopic peak in an isotopic pattern (0 = disabled); see the definition of [Isotope m/z Tolerance](@ref settings_mzdifftolerance).

<p><b>Isotope Intensity Tolerance</b> - the maximum tolerance of intensities of matched isotopes (e.g. 10%); see the definition of [Isotope Intensity Tolerance](@ref settings_intensitytolerance).</p>

<p>
\image html tutorial-2-3-settings.png "Tutorial 3: Configuration for identification of unknown compounds in LC-MS data."
\image latex tutorial-2-3-settings.png "Tutorial 3: Configuration for identification of unknown compounds in LC-MS data." width=16cm
</p>

<p>Similarly to the [Tutorial 2](@ref tutorial2_cb2), the compounds can be browsed in the [Summary Table of Matched Peaks](@ref summarypeakstable) while their extracted ion chromatograms can be directly seen in the [Chromatogram](@ref chromatogram) window.
The difference is that the formulas of compounds are automatically suggested instead of the names of compounds.</p>

<p>
\image html tutorial-2-3-summarytable.png "Tutorial 3: Summary Table of Matched Peaks."
\image latex tutorial-2-3-summarytable.png "Tutorial 3: Summary Table of Matched Peaks." width=16cm
</p>

\section tutorial4_cb2 Tutorial 4: Dereplication in MSI data and fusion with histology image

<p>Follow the instructions in [Tutorial 1](@ref tutorial1_cb2) and [Tutorial 2](@ref tutorial2_cb2). Modify the following parameters:</p>

<p><b>File</b> - select an input imzML file.</p>

<p><b>Minimum Number of Spectra</b> - the minimum number of single-pixel mass spectra in which a compound must be matched to be reported.</p>

<p>
\image html tutorial-2-4-settings.png "Tutorial 4: Configuration for dereplication in MSI data."
\image latex tutorial-2-4-settings.png "Tutorial 4: Configuration for dereplication in MSI data." width=16cm
</p>

<p>Every row in the output report includes coordinates [x, y] and corresponds to a single-pixel mass spectrum. Double-click on any row to display the spectrum from the corresponding pixel.</p>

<p>
\image html tutorial-2-4-result.png "Tutorial 4: Search process completed."
\image latex tutorial-2-4-result.png "Tutorial 4: Search process completed." width=16cm
</p>

<p>In the main window, click on [Tools->CrossVis](@ref crossvis). In the [CrossVis](@ref crossvis) tool, click on <b>File->Open</b>, select <b>Optical Image</b>, and click <b>Ok</b>. Then select the image file which was used during the acquisition of your data.
The optical image is automatically correlated with pixels (or squares) corresponding to the individual mass spectra. In some cases, the correlation of data may be vendor specific, see the details [here](@ref subsec_crossvisvendordetails).</p>

<p>
\image html tutorial-2-4-imagewindow.png "Tutorial 4: Fusion of optical image with identified compounds."
\image latex tutorial-2-4-imagewindow.png "Tutorial 4: Fusion of optical image with identified compounds." width=16cm
</p>

<p>The color of every square represents the sum of absolute/relative intensities of all matched peaks in the correspoding mass spectrum by default. 
However, the [CrossVis](@ref crossvis) tool is interactively-connected with [Summary Table of Matched Peaks](@ref summarypeakstable). To display a single compound or to browse spatial distributions of all identified compounds,
open the [Tools->Summary Table of Matched Peaks](@ref summarypeakstable) and filter the list of all matched peaks by the name of a compound or by theoretical m/z ratio.
Start typing into the text field and the names of identified compounds are automatically suggested. Use the arrows to browse the identified compounds individually.</p>

<p>
\image html tutorial-2-4-summarytable.png "Tutorial 4: Summary Table of Matched Peaks."
\image latex tutorial-2-4-summarytable.png "Tutorial 4: Summary Table of Matched Peaks." width=16cm
</p>

<p>If you would like to make a fusion with another image (e.g. a histology image), click on <b>File->Open</b>, select <b>Histology Image</b>, and click <b>Ok</b>. Then select an image in *.jpg, *.jpeg, *.png, *.tif, *.tiff, *.bmp or *.gif file format.
Click on <b>View->Show Selection</b> to see a bounding box around the opened histology image. Make sure that <b>Histology Image</b> layer is selected in the list of layers. The histology image must be correlated with optical image manually.
To make the fusion, move the histology image by mouse to overlap the analyzed sample on the optical image. If needed, adjust the size and rotation angle of the histology image on the corresponding toolbar (the active toolbar is green).
Optionally, adjust the transparency and order of any layer on the right panel. See more details [here](@ref crossvis).</p>

<p>
\image html crossvis-histology.png "Tutorial 4: fusion of histology image (white) with an optical image (black)."
\image latex crossvis-histology.png "Tutorial 4: fusion of histology image (white) with an optical image (black)." width=16cm
</p>
