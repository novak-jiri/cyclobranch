\subsection subsec_sequenceeditorcolumns Columns

<p>
<ul>
<li> Type - The type of a peptide (linear, cyclic, branched, branch-cyclic, linear-polyketide or cyclic-polyketide).
<li> Name - Name of a peptide.
<li> Summary Formula - Summary formula of a neutral molecule.
<li> Monoisotopic Mass - Monoisotopic mass of the peptide. The value is recalculated when the summary formula is changed.
<li> Sequence - Peptide sequence. If the column is empty, the database can be utilized for identification of compounds in MS spectra. If the column is non-empty, the database can be used in both MS and MS/MS mode.
<li> N-terminal Modification - A name of an N-terminal modification of a linear or a branched peptide.
<li> C-terminal Modification - A name of a C-terminal modification of a linear or a branched peptide.
<li> Branch Modification - A name of an N-terminal or a C-terminal modification of a branch of a branched or a branch-cyclic peptide.
<li> Reference - Reference id of a peptide. See [Format of Sequence Databases](@ref sequenceeditordbformat).
<li> View - View the molecule structure in a reference database (ChemSpider, PubChem or Norine).
</ul>
</p>

\section sequenceeditordbformat Format of Sequence Databases

<p>The database of sequences is a text file (*.txt) containing a list of sequences (one sequence per line). Each line contains the following items separated by tabulars: 
<ul>
<li> type of a sequence (linear, cyclic, branched, branch-cyclic, linear-polyketide or cyclic-polyketide)
<li> name of a peptide
<li> summary formula of a neutral molecule
<li> monoisotopic mass of peptide (the value is ignored; the mass is automatically calculated from the summary formula)
<li> a peptide sequence
<li> name of an N-terminal modification
<li> name of a C-terminal modification
<li> name of a branch modification
<li> reference id of a peptide
</ul>

A line starting with the hash '#' is a comment which is not processed.
</p>

<p>
The format of a peptide sequence differs according to a peptide type:
<ul>
<li> <b>Linear peptide:</b> <code>[aaa/bbb]-[ccc]-[ddd]-[eee]-[fff]</code>. The names of building blocks must be enclosed in <code>[</code> and <code>]</code>. The building blocks are separated by <code>-</code>. Isomers of a building block are separated by <code>/</code>.
<li> <b>Cyclic peptide:</b> the same syntax like in case of a linear peptide.
<li> <b>Branched peptide:</b> <code>[aaa/bbb]\\([ccc]-[ddd]-[eee]\\)[fff]</code>. The syntax is similar to a linear peptide. The string <code>\\(</code> is used instead of <code>-</code> to determine the start of a branch. The string <code>\\)</code> is used instead of <code>-</code> to determine the end of a branch.
The building block <code>[ccc]</code> is the block where the branching occurs. The building block <code>[eee]</code> is the terminal block of the branch. See also [Branched Peptides](@ref brancheddetail).
<li> <b>Branch-cyclic peptide:</b> the same syntax like in case of a branched peptide.
<li> <b>Linear polyketide:</b> the same like in case of a linear peptide (acronyms of building blocks eliminating 2 hydrogens can be included in the sequence).
<li> <b>Cyclic polyketide:</b> the same like in case of a cyclic peptide (acronyms of building blocks eliminating 2 hydrogens can be included in the sequence).
</ul>
</p>

<p>
Reference ids can be written in the following formats (without quotes):
<ul>
<li> "CSID: <number>" (ChemSpider ID; example: "CSID: 13539649")
<li> "CID: <number>" (PubChem ID; example: "CID: 12439044")
<li> "CHEBI: <number>" (ChEBI ID; example: "CHEBI: 27266")
<li> "DOI: <doi>" (example: "DOI: 10.1016/j.bbapap.2016.12.003")
<li> "<norine_id>" (Norine ID; example: "NOR00205")
<li> "<lipidmaps_id>" (LipidMaps ID; example: "LMFA00000001")
</ul>
</p>
