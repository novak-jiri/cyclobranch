\subsection subsec_blockseditorcolumns Columns

<p>
<ul>
<li> Name(s) - Name of a building block (or more blocks corresponding to isomers separated by '/').
<li> Acronym(s) - Acronym of a building block (or more blocks corresponding to isomers separated by '/').
<li> Residue Summary - Residue summary formula of a building block.
<li> Monoisotopic Residue Mass - Monoisotopic mass corresponding to a block. The value is recalculated when the summary formula is changed.
<li> Reference(s) - Reference id of a building block (or more blocks corresponding to isomers separated by '/'). See [Format of Building Blocks Databases](@ref blockseditordbformat).
<li> Preview - A link to a reference database (e.g., ChemSpider) generated from the fields acronym and reference. The link is generated automatically and cannot be edited manually.
</ul>

The numbers of '/' must be equal in a row in the fields Name, Acronym and Reference.
The names and acronyms must not contain the characters '[' or ']'.
</p>

\section blockseditordbformat Format of Building Block Databases

<p>The database of building blocks is a text file (*.txt) containing a list of building blocks (one building block per line). Each line contains the following items separated by tabulars: 
<ul>
<li> name of a block (or more blocks corresponding to isomers separated by '/')
<li> acronym of a block (or more blocks corresponding to isomers separated by '/')
<li> residue summary formula of a block (important: an exact mass of the block is calculated from this value)
<li> mass corresponding to the block (the value is ignored; the mass is automatically calculated from the residue summary formula)
<li> reference id (or more blocks corresponding to isomers separated by '/')
</ul>

The numbers of '/' must be equal on a line in the fields name, acronym and reference id.
The names and acronyms must not contain the characters '[' or ']'.
A line starting with the hash '#' is a comment which is not processed.
</p>

<p>
Reference ids can be written in the following formats (without quotes):
<ul>
<li> "CSID: <number>" (ChemSpider ID; example: "CSID: 13539649")
<li> "CID: <number>" (PubChem ID; example: "CID: 12439044")
<li> "PDB: <pdb_code>" (PDB code; example: "PDB: UAL")
<li> "<smiles> in CSID: <number>" (smiles in ChemSpider ID; example: "C(C(CN)NC(C)=O)(=O)O in CSID: 8570696")
<li> "<smiles> in CID: <number>" (smiles in PubChem ID; example: "C1=C(C(=CC2=C1N3C(C(=C2)N)NC(CC3)C(O)=O)O)O in CID: 5289234")
<li> "<smiles> in: <norine_id>" (smiles in Norine; example: "C1=C(C(=CC2=C1N3C(C(N)C2)NCCC3C(O)=O)O)O in: NOR00903")
</ul>
</p>

<hr>

\section ketideblocks Format of Ketide Building Blocks

<p>
The identification of linear and cyclic polyketides requires an additional annotation of building blocks. A name of a building block eliminating 2 hydrogens instead of the water molecule must start with the string "(-2H) " without quotes. 
Corresponding residue summary formulas must be entered in the column "Residue Summary" (i.e., 2 hydrogens or the water molecule must be deducted manually).
</p>
